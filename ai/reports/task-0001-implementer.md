## Assumptions

1. **`moduleResolution: Bundler` (not `NodeNext`)** — The spec states `module: NodeNext` in `tsconfig.base.json`, but Cloudflare Workers require Bundler resolution. More critically, `Bundler` enables TypeScript to resolve package types directly from source (`"types": "./src/index.ts"` in exports), which is necessary for `turbo run typecheck` with `"dependsOn": ["^typecheck"]` (no prior build step required).

2. **Worker build script** — Used `wrangler deploy --dry-run --outdir=dist` instead of the deprecated `wrangler build`. This is the current Cloudflare-recommended equivalent.

3. **`@cloudflare/workers-types` in all packages** — Added to all packages' devDependencies and tsconfig `types` arrays. The `packages/types` package defines the `Env` interface using CF-specific globals; all downstream packages need those types in scope for typecheck to pass.

4. **Lint deferred** — ESLint is not wired in this scaffold. Each package has `"lint": "echo 'lint deferred'"` so `turbo run lint` exits 0. This is intentional per acceptance criteria.

## Remaining Gaps

1. **Lint not wired** — ESLint + `@typescript-eslint` not configured. Should be added in a follow-up or in Task 0002.
2. **`kiox` smoke test not run** — `kiox` binary is not available in the local environment. This will be validated in CI via `kiox-action@v2.1.2`.
3. **Wrangler 3.x deprecation warning** — Wrangler 3 prints "out-of-date" warnings; wrangler 4 is available. The spec pins `^3.65.0` so this is intentional. Can be upgraded in a follow-up.
4. **No `.nvmrc` or `engines` enforcement** — Node 20 is declared in `engines` fields but not enforced via `.nvmrc`.
5. **No test files** — All tests pass via `passWithNoTests: true`. Real tests will be added starting Task 0003.
6. **`esbuild`/`workerd` build scripts** — pnpm requires explicit approval for these. The `.npmrc` file sets `approve-builds[]` but pnpm 10.12.1 appears to need interactive `pnpm approve-builds` for persistent config. In CI, `kiox-action` handles workspace setup; this only affects fresh local installs.
