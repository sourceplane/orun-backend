## Fixture Fix

1. Added `compositions.sources` pointing to a local `./compositions` directory.
2. Created `compositions/stack.yaml` (required Stack manifest for source auto-discovery).

Resulting DAG (6 jobs):
```
foundation@dev.smoke      (no deps)
foundation@stage.smoke    (no deps)
api@dev.smoke             depends on foundation@dev.smoke
api@stage.smoke           depends on foundation@stage.smoke
web@dev.smoke             depends on api@dev.smoke
web@stage.smoke           depends on api@stage.smoke
```

## Matrix CI Example

`examples/github-actions/remote-state-matrix.yml` demonstrates:

- `permissions: id-token: write` for GitHub Actions OIDC
- Plan compiled once, uploaded as artifact
- Matrix fan-out with one runner per job ID
- Duplicate job entry appended to prove idempotent claim behavior
- Shared deterministic `ORUN_EXEC_ID` across all matrix children
- Each child runs exactly one job with `--job`
- Environment fan-out jobs (`--env dev`, `--env stage`) run independently
- Verification job asserts all jobs reached success via `orun status --remote-state --json`
- Remote log retrieval via `orun logs --remote-state --job <id>`

3. The conformance workflow is gated by `vars.REMOTE_STATE_TESTS == 'true'` to avoid hitting the backend on routine pushes.
4. `orun logs --remote-state` returns step-level output uploaded during execution (currently returns summary + step markers; full streaming log upload depends on runner step-log capture which works for the smoke steps).

## Remaining Gaps

1. **Step-level log granularity** — Remote logs return the aggregated output per job. Per-step log segmentation in the backend (stored as separate step entries) works but the CLI displays them in a flattened view. This is a UX polish item, not a correctness issue.
2. **Heartbeat verification** — The conformance workflow does not explicitly assert heartbeat behavior (jobs are too short for heartbeat timeout). A long-running job fixture could test this in the future.
3. **Cancellation testing** — The conformance workflow does not test `orun run --cancel` against remote state. This could be added as a separate conformance case.
