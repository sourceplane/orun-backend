## 2. Cloudflare Worker Deployment Contract

Make `cloudflare-worker-turbo` and, where applicable, `cloudflare-worker` suitable for live deployment of a Worker with Durable Objects, D1, R2, cron triggers, migrations, and secrets.

Required behavior:

- Accept either `wrangler.jsonc` or `wrangler.toml` for Worker structure validation.
- Fail clearly when a production deploy is attempted without Cloudflare credentials.
- Validate Cloudflare auth before attempting a production deploy.
- Support explicit pre-deploy/bootstrap hooks for consumer repos that need resource provisioning.
- Support D1 migrations as a first-class deploy step or as a clearly documented hook.
- Make logs unambiguous:
  - dry-run
  - skipped because non-production/non-production branch
  - live deployment attempted
  - live deployment succeeded
- Avoid a false green when the deploy command did not run.
- Ensure examples and docs explain how a consumer repo should handle D1 database IDs and R2 buckets.

Preferred composition input shape:

```yaml
inputs:
  nodeVersion: "20"
  pnpmVersion: "10.12.1"
  productionBranch: main
  wranglerConfig: wrangler.jsonc
  preDeployCommand: "..."
  migrationCommand: "..."
  deployCommand: "pnpm run deploy"
  smokeCommand: "..."
```

You may choose a different shape if it is cleaner, but it must solve the `orun-backend` use case without custom GitHub Actions.

Do not hard-code `orun-backend` values in stack-tectonic. This must remain a reusable stack composition.
