## Routes

Add these routes in `apps/worker/src/router.ts` with session authentication:

```text
POST   /v1/accounts
GET    /v1/accounts/me
POST   /v1/accounts/repos
GET    /v1/accounts/repos
DELETE /v1/accounts/repos/:namespaceId
```

Expected behavior:

- `POST /v1/accounts`
  - Requires a session JWT.
  - Empty JSON body or no body is acceptable.
  - Idempotently creates or returns the account for `authCtx.actor` / session `sub`.
  - Response:

    ```json
    { "accountId": "uuid", "githubLogin": "octocat", "createdAt": "ISO8601" }
    ```

- `GET /v1/accounts/me`
  - Requires a session JWT.
  - Returns the existing account for the session login.
  - If no account exists yet, return a typed JSON 404 (`NOT_FOUND`) rather than implicitly creating it. Creation belongs to `POST /v1/accounts`.

- `POST /v1/accounts/repos`
  - Requires a session JWT.
  - Requires body `{ "repoFullName": "owner/repo" }`.
  - Requires a short-lived GitHub access token in `X-GitHub-Access-Token`.
  - Uses that token only for GitHub admin verification.
  - Never stores, logs, echoes, or embeds the token in errors.
  - Ensures the account exists before linking.
  - Upserts `namespaces` with the GitHub repository ID and full slug before inserting into `account_repos`.
  - Idempotently inserts the link.
  - Response:

    ```json
    { "namespaceId": "123456789", "namespaceSlug": "owner/repo", "linkedAt": "ISO8601" }
    ```

- `GET /v1/accounts/repos`
  - Requires a session JWT.
  - If the account does not exist yet, return `{ "repos": [] }`.
  - Joins `account_repos` to `namespaces`.
  - Orders by `linked_at DESC`.
  - Response:

    ```json
    { "repos": [{ "namespaceId": "123456789", "namespaceSlug": "owner/repo", "linkedAt": "ISO8601" }] }
    ```

- `DELETE /v1/accounts/repos/:namespaceId`
  - Requires a session JWT.
  - Deletes only the account visibility link.
  - Does not delete namespaces, runs, jobs, logs, or coordinator state.
  - Be idempotent. If the account or link is absent, still return success.
  - Response may be `204 No Content` or JSON `{ "ok": true }`; prefer JSON if that better matches existing handlers.

## GitHub Admin Verification

Implement and export a testable function similar to:

```typescript
verifyRepoAdminAccess(githubLogin, repoFullName, githubAccessToken, fetchImpl?)
```

Expected behavior:

- Validate `repoFullName` strictly as `owner/repo`.
  - No empty owner or repo.
  - No leading or trailing slash.
  - No extra slash.
  - No traversal segments.
  - URL-encode path segments with `encodeURIComponent`.

- Call:

  ```text
  GET https://api.github.com/repos/{owner}/{repo}
  ```

- Required GitHub headers:

  ```text
  Authorization: Bearer <token>
  Accept: application/vnd.github+json
  User-Agent: orun-backend-account-linking
  ```

- Parse at least:
  - `id`
  - `full_name`
  - `owner.login`
  - `permissions.admin`

- If repo response is `404`, throw `NOT_FOUND`.
- If repo response is non-OK, throw `INTERNAL_ERROR`.
- If `permissions.admin === true`, linking is allowed.
- If repo admin is false, check org admin:

  ```text
  GET https://api.github.com/orgs/{owner}/memberships/{githubLogin}
  ```

- Treat org admin success as `role === "admin"`.
- If org membership lookup is non-OK or role is not admin, throw `FORBIDDEN`.
- For user-owned repositories, the org fallback may not apply; non-admin should remain `FORBIDDEN`.
- Missing token should produce a typed client error (`INVALID_REQUEST` or `UNAUTHORIZED`). Pick one, apply consistently, and test it.
- Do not call live GitHub in tests. Mock `fetch`.

Return the verified repository metadata needed for D1 storage:

```typescript
{
  namespaceId: String(repo.id),
  namespaceSlug: repo.full_name
}
```
