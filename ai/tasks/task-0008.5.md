## 2. Fix Example GitHub Actions Matrix Workflow

```text
examples/github-actions/remote-state-matrix.yml
```

Required behavior:

- Uses GitHub Actions OIDC, not `ORUN_TOKEN`, for the live backend path.
- Has:

  ```yaml
  permissions:
    contents: read
    id-token: write
  ```

- Uses `vars.ORUN_BACKEND_URL`.
- Compiles the plan once.
- Uploads the compiled `.orun/plans/` content as an artifact.
- Matrix children download the plan artifact.
- Matrix includes one entry per plan job plus at least one duplicate job entry.
- Matrix children share the same deterministic `ORUN_EXEC_ID`.
- Each matrix child runs exactly one selected job with `--job`.
- Duplicate job runner exits cleanly when the job is already running or complete.
- Dependency-waiting jobs poll remote state instead of failing because local state is empty.
- Includes environment fanout examples:

  ```bash
  orun run <planID> --env dev --remote-state
  orun run <planID> --env stage --remote-state
  ```

- Final verification job runs:

  ```bash
  orun status --remote-state --exec-id <runID> --backend-url <url> --json
  orun logs --remote-state --exec-id <runID> --backend-url <url> --job <jobID>
  ```

- Final verification asserts all expected jobs reached success.
