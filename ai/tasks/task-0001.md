# Acceptance Criteria
The PR is acceptable when all of these are true:

1. The required file/directory structure exists.
2. `pnpm install` succeeds and commits `pnpm-lock.yaml`.
3. `pnpm exec turbo run typecheck` succeeds.
4. `pnpm exec turbo run build` succeeds, or any Cloudflare-specific limitation is clearly documented with the smallest possible follow-up.
5. `pnpm exec turbo run test` succeeds with placeholder/no-op tests or clean Vitest config.
6. `pnpm exec turbo run lint` succeeds if linting is wired; otherwise explain why lint is intentionally deferred.
7. `apps/worker/wrangler.jsonc` has the required bindings and vars.
8. `intent.yaml`, `kiox.yaml`, and every required `component.yaml` exist and match the tectonic model.
9. `.github/workflows/workflow.yml` uses `sourceplane/kiox-action@v2.1.2` and runs `kiox -- orun plan`/`kiox -- orun run --execute --gha` as specified.
10. No real backend business logic is introduced.
11. A GitHub PR is opened against `sourceplane/orun-backend:main`.

Optional smoke checks, if the tools are installed:

- `cd apps/worker && pnpm exec wrangler deploy --dry-run`
- `kiox -- orun plan`

If these tools are not installed or cannot run locally, record that clearly in the report.
