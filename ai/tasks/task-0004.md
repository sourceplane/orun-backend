## R2Storage

Implement:

```typescript
export class R2Storage {
  constructor(private bucket: R2Bucket) {}

  async writeLog(
    namespaceId: string,
    runId: string,
    jobId: string,
    content: string | ReadableStream,
    options?: { expiresAt?: string | Date }
  ): Promise<string>;

  async readLog(namespaceId: string, runId: string, jobId: string): Promise<R2ObjectBody | null>;

  async savePlan(namespaceId: string, plan: Plan): Promise<string>;

  async getPlan(namespaceId: string, checksum: string): Promise<Plan | null>;

  async listRunLogs(namespaceId: string, runId: string): Promise<string[]>;

  async deleteRun(namespaceId: string, runId: string): Promise<void>;
}
```

Required behavior:

- Use `runLogPath(namespaceId, runId, jobId)` from `@orun/types/paths`.
- Use `planPath(namespaceId, plan.checksum)` from `@orun/types/paths`.
- `writeLog` stores plain text logs with `Content-Type: text/plain; charset=utf-8`.
- If `options.expiresAt` is provided, set R2 custom metadata `{ "expires-at": isoString }`.
- Return the R2 key from `writeLog`.
- `readLog` returns the R2 object body or `null`.
- `savePlan` stores immutable plan JSON with `Content-Type: application/json; charset=utf-8` and returns the R2 key.
- `getPlan` returns parsed `Plan` or `null` if missing.
- `listRunLogs` lists keys under `${namespaceId}/runs/${runId}/logs/` and handles paginated R2 results.
- `deleteRun` deletes all objects under `${namespaceId}/runs/${runId}/` and handles pagination.
- All R2 access must be namespace-scoped. Never list or delete across namespace boundaries.

## D1Index

Implement:

```typescript
export class D1Index {
  constructor(private db: D1Database) {}

  async upsertNamespace(namespace: Namespace): Promise<void>;

  async createRun(run: Run): Promise<void>;

  async updateRun(
    namespaceId: string,
    runId: string,
    update: Partial<Pick<Run, "status" | "jobDone" | "jobFailed" | "finishedAt" | "updatedAt">>
  ): Promise<void>;

  async listRuns(namespaceIds: string[], limit?: number, offset?: number): Promise<Run[]>;

  async getRun(namespaceId: string, runId: string): Promise<Run | null>;

  async upsertJob(job: IndexedJobInput): Promise<void>;

  async listJobs(namespaceId: string, runId: string): Promise<Job[]>;

  async deleteExpiredRuns(now?: string | Date): Promise<number>;
}
```

Define and export a package-local input type because `Job` from `@orun/types` does not include `namespaceId`, while the D1 schema requires it:

```typescript
export type IndexedJobInput =
  Pick<Job, "jobId" | "runId" | "component" | "status" | "runnerId" | "startedAt" | "finishedAt" | "logRef"> & {
    namespaceId: string;
  };
```

Required behavior:

- `upsertNamespace` uses `namespace_id` as the immutable key and updates mutable `namespace_slug`/`last_seen_at`.
- `createRun` upserts the namespace and inserts or idempotently upserts the run row. Preserve namespace isolation.
- `updateRun` only updates the allowed fields and never constructs SQL from untrusted field names.
- `listRuns`:
  - returns `[]` for an empty namespace list.
  - filters by `namespace_id IN (...)`.
  - orders by `created_at DESC`.
  - supports sane `limit` and `offset` defaults.
- `getRun` filters by both `namespace_id` and `run_id`.
- `upsertJob` inserts or updates the row keyed by `(namespace_id, run_id, job_id)`.
- `listJobs` filters by both namespace and run.
- `deleteExpiredRuns` deletes jobs and runs for expired runs where `expires_at <= now` and returns the number of run rows deleted.
- Do not delete `namespaces`, `accounts`, or `account_repos` in `deleteExpiredRuns`.
- All D1 queries must include namespace filters where a namespace-scoped operation is intended.

Contract gap to handle deliberately:

- The D1 `jobs` table does not store `deps`, `lastError`, or `heartbeatAt`, but `Job` requires those fields. For `listJobs`, return `deps: []`, `lastError: null`, and `heartbeatAt: null`, and document this as derived-index lossiness in the implementer report. Do not add coordination-only fields to D1 unless you can justify a schema change from the specs.
