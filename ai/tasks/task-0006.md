## Runs API

### `POST /v1/runs`

Required behavior:

- Authenticate OIDC or session.
- Resolve target namespace.
- Apply rate limit for the target namespace.
- Parse and validate JSON body.
- Require a valid `plan`.
- Use `body.runId` when supplied; otherwise generate `crypto.randomUUID()`.
- Reject empty or non-string `runId` if supplied.
- Call coordinator `POST /init` with:

```typescript
{
  plan,
  runId,
  namespaceId: namespace.namespaceId,
  namespaceSlug: namespace.namespaceSlug
}
```

- If the coordinator already exists, verify the existing coordinator state's plan checksum matches the submitted plan checksum. If it differs, return `CONFLICT`.
- Save the plan to R2 with `R2Storage.savePlan(namespaceId, plan)`.
- Mirror the run and initial jobs into D1 using `D1Index`.
- D1/R2 mirrors are not authoritative. It is acceptable to use `ctx.waitUntil(...)` for non-critical mirrors, but tests must be able to flush or await those promises deterministically.
- Return:

```json
{
  "runId": "...",
  "status": "running",
  "createdAt": "ISO8601"
}
```

- Use `201` for a newly initialized run.
- Use `200` for an idempotent join to an existing run.

Run row defaults:

- `status`: `running`
- `planChecksum`: `plan.checksum`
- `triggerType`: body value or `ci`
- `actor`: body actor or `authCtx.actor`
- `dryRun`: boolean body value or `false`
- `jobTotal`: `plan.jobs.length`
- `jobDone`: `0`
- `jobFailed`: `0`
- `expiresAt`: default 24 hours after creation

Also upsert one D1 job row per plan job with `pending` status and null runner/timestamps/log.

### `GET /v1/runs`

Required behavior:

- Session auth only.
- Apply a bounded rate limit. Until Task 0007 adds account scoping, use a stable session key or apply namespace limits across `allowedNamespaceIds`; document the choice.
- Support optional `limit` and `offset` query params with safe bounds.
- List recent runs with `D1Index.listRuns(authCtx.allowedNamespaceIds, limit, offset)`.
- Return:

```json
{ "runs": [ ... ] }
```

### `GET /v1/runs/:runId`

Required behavior:

- OIDC or session.
- Resolve and enforce namespace access.
- Prefer the Coordinator DO's `/state` for fresh execution state when the namespace is known.
- Fall back to `D1Index.getRun()` only if the coordinator state is unavailable and the index has a row.
- Return `NOT_FOUND` if neither source has the run.

Response may be either a `Run` object or `{ "run": Run }`, but be consistent and test it.

## Jobs API

### `POST /v1/runs/:runId/jobs/:jobId/claim`

- OIDC only.
- Rate limit by OIDC namespace.
- Forward `{ runnerId }` to coordinator `/jobs/:jobId/claim`.
- Return coordinator JSON with status `200` for both claimed and unclaimed results.

### `POST /v1/runs/:runId/jobs/:jobId/update`

- OIDC only.
- Rate limit by OIDC namespace.
- Require `{ runnerId, status, error? }`.
- Forward the full `CoordinatorUpdateJobRequest` to coordinator `/jobs/:jobId/update`.
- After a successful coordinator update, mirror the coordinator state into D1 via `ctx.waitUntil(...)`:
  - Update run status/progress counters.
  - Upsert jobs with current status/runner/timestamps.
- Do not make D1 authoritative for execution decisions.

### `POST /v1/runs/:runId/jobs/:jobId/heartbeat`

- OIDC only.
- Rate limit by OIDC namespace.
- Require `{ runnerId }`.
- Forward to coordinator `/jobs/:jobId/heartbeat`.
- Return coordinator JSON.

### `GET /v1/runs/:runId/runnable`

- OIDC only.
- Rate limit by OIDC namespace.
- Forward to coordinator `/runnable`.
- Return coordinator JSON.

### `GET /v1/runs/:runId/jobs`

- OIDC or session.
- Resolve and enforce namespace access.
- Prefer coordinator `/state` and map jobs to the public `Job` shape for fresh status.
- Fall back to `D1Index.listJobs(namespaceId, runId)` when coordinator state is unavailable.
- Return:

```json
{ "jobs": [ ... ] }
```

### `GET /v1/runs/:runId/jobs/:jobId/status`

- OIDC or session.
- Resolve and enforce namespace access.
- Prefer coordinator `/jobs/:jobId/status`.
- Return `NOT_FOUND` if the coordinator has no state/job and D1 fallback cannot find the job.

## Logs API

### `POST /v1/runs/:runId/logs/:jobId`

- OIDC only.
- Rate limit by OIDC namespace.
- Accept text bodies and streamed request bodies.
- Write to R2 with `R2Storage.writeLog(namespaceId, runId, jobId, request.body ?? "", { expiresAt })`.
- Use the run's D1 `expiresAt` when available; otherwise default log expiry to 24 hours after upload.
- Attach the returned `logRef` to the indexed job row without corrupting existing job status. If needed, fetch coordinator job status first, then upsert the job row with `logRef`.
- Return:

```json
{ "ok": true, "logRef": "..." }
```

### `GET /v1/runs/:runId/logs/:jobId`

- OIDC or session.
- Resolve and enforce namespace access.
- Read with `R2Storage.readLog(namespaceId, runId, jobId)`.
- Return `NOT_FOUND` JSON if missing.
- Stream the object body as `text/plain; charset=utf-8` when present.
- Include CORS headers.

## Scheduled Worker
