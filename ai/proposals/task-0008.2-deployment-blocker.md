# Proposed Spec Change

Treat live deployment as a release gate before Dashboard UI.

1. Insert Task 0008.2 to update `sourceplane/stack-tectonic` so the Cloudflare Worker and Pages compositions are actually usable for Sourceplane repositories.
2. Release the updated stack as a new OCI version.
3. Insert Task 0008.3 to consume the new stack in `sourceplane/orun-backend`, provision Cloudflare resources, deploy the Worker, apply migrations, configure secrets/vars, and verify the live API.
4. Keep Task 0009 Dashboard UI blocked until Task 0008.3 verifier confirms live deploy.
5. Update `spec/02-devops.md` and `SCHEDULE.md` so future tasks distinguish:
   - PR validation
   - dry-run deploys
   - real production deployments
   - live endpoint verification
6. Require CI logs/reports to show nonzero production deploy jobs or an explicit intentional no-op. A successful run with `0 components x 3 envs -> 0 jobs` is not deployment evidence.

# Why This Is Needed

The backend cannot support dashboard or remote-state users if the Worker, D1, R2, migrations, and secrets are not live.

Green CI has been proving build/typecheck/dry-run behavior, not Cloudflare production availability. Continuing to Dashboard UI would compound the gap by building a UI against an API that is not actually deployed.
